## Citation Standards

### Academic Citation Requirements

#### Mandatory Standards
- **Minimum 50% peer-reviewed sources** across all chapters
- **APA 7th edition formatting** for all citations
- **In-text citations** for all technical claims
- **Reference list completeness** with no missing information
- **Consistent citation style** throughout the book

#### Peer-Reviewed Source Verification
- Publications from established academic presses (MIT Press, Springer, etc.)
- Papers published in recognized journals (IEEE, ACM, etc.)
- Conference papers from top-tier venues (RSS, ICRA, IROS, etc.)
- Preprints from arXiv.org with high citation counts and community validation

#### Academic Quality Assessment
- **Author Affiliation**: Authors from recognized institutions (universities, research labs)
- **Citation Count**: Minimum 5 citations or publication in high-impact venues
- **Recency**: Preference for papers published within last 10 years for rapidly evolving fields
- **Relevance**: Direct application to Physical AI and humanoid robotics

### APA 7th Edition Formatting Standards

#### Journal Articles
```
Author, A. A., Author, B. B., & Author, C. C. (Year). Title of article. Title of Periodical, volume(issue), pages. https://doi.org/xx.xxxx/xxxxx
```

#### Conference Papers
```
Author, A. A., & Author, B. B. (Year, Month). Title of paper. In Proceedings of the Conference Name (pp. page-page). Publisher. https://doi.org/xx.xxxx/xxxxx
```

#### Books
```
Author, A. A. (Year). Title of work: Capital letter also for subtitle. Publisher Name.
```

#### Technical Reports
```
Author, A. A., & Author, B. B. (Year). Title of report (Report No. Report Number). Publisher/Institution.
```

#### Software Documentation
```
Project/Company Name. (Year). Title of documentation or tool. Publisher. URL
```

#### Preprints
```
Author, A. A., Author, B. B., & Author, C. C. (Year). Title of paper [Preprint]. arXiv. https://arxiv.org/abs/xxxx.xxxxx
```

### Citation Integration Standards

#### In-Text Citation Placement
- Parenthetical citations: (Author, Year) or (Author, Year, p. X)
- Narrative citations: Author (Year) showed that...
- Multiple citations: (Author1, Year; Author2, Year)
- Page-specific citations: (Author, Year, pp. X-Y)

#### Citation Context Requirements
- **Support Statements**: Every technical claim must be supported
- **Introduction Context**: Citations for foundational concepts
- **Comparison Context**: Citations for alternative approaches
- **Application Context**: Citations for practical implementations

#### Reference List Organization
- Alphabetically ordered by first author's last name
- Double-spaced throughout
- Hanging indent for second and subsequent lines
- Complete bibliographic information for all sources

### Academic Integrity Standards

#### Plagiarism Prevention
- All sources properly attributed
- Direct quotes in quotation marks
- Paraphrased content properly cited
- Ideas from sources properly credited

#### Originality Requirements
- Synthesis of multiple sources for new insights
- Critical analysis of existing literature
- Original connections between concepts
- Distinct contribution to field knowledge

#### Self-Citation Policy
- Limited to previously published work of authors
- Justified by direct relevance to topic
- Not excessive in any single chapter
- Properly distinguished from other sources

## Diagram Standards

### SpeckitPlus Integration Standards

#### Diagram Generation Process
1. **Prompt Creation**: Use PHR (Prompt Helper Record) system for consistency
2. **Style Consistency**: Follow established visual language
3. **Technical Accuracy**: Diagrams must accurately represent concepts
4. **Quality Validation**: Verify diagram quality before inclusion

#### Standard Diagram Categories

##### 1. Architecture Diagrams
- **Purpose**: Show system structure and component relationships
- **Style**: Clean, hierarchical representation
- **Elements**: 
  - Components in boxes with clear labels
  - Relationships with arrows showing data/control flow
  - Color coding for different system types
  - Legends when needed

##### 2. Process Flow Diagrams
- **Purpose**: Show algorithm steps or process sequences
- **Style**: Flowchart with decision points
- **Elements**:
  - Process steps in rectangles
  - Decisions in diamonds
  - Start/end points in circles
  - Arrows showing flow direction

##### 3. Conceptual Diagrams
- **Purpose**: Illustrate abstract concepts and relationships
- **Style**: Clear visual representation of ideas
- **Elements**:
  - Conceptual elements with clear labels
  - Relationships with appropriate connecting lines
  - Visual metaphors when helpful
  - Color to distinguish concepts

##### 4. Technical Data Diagrams
- **Purpose**: Show technical specifications, parameters, or measurements
- **Style**: Precise and data-focused
- **Elements**:
  - Axes and scales when needed
  - Data points or ranges clearly marked
  - Technical annotations
  - Units of measurement

### Visual Language Standards

#### Color Palette
- **Primary Blue**: #1f77b4 (system components)
- **Secondary Orange**: #ff7f0e (data flows)
- **Accent Green**: #2ca02c (success/positive elements)
- **Warning Red**: #d62728 (errors/warnings)
- **Neutral Gray**: #7f7f7f (supporting elements)

#### Typography
- **Headings**: Sans-serif, bold, clearly hierarchical
- **Body Text**: Sans-serif, readable in diagrams
- **Labels**: Consistent size and positioning
- **Legends**: Clear and positioned for readability

#### Layout and Composition
- **Consistent Margins**: Standard spacing around diagrams
- **Alignment**: Left-aligned or center-aligned consistently
- **Hierarchy**: Visual importance through size and position
- **White Space**: Proper spacing between elements

### Technical Accuracy Requirements

#### Scientific Accuracy
- All diagrams must represent actual technical concepts
- Mathematical relationships correctly depicted
- Physical principles accurately represented
- No misleading simplifications

#### Engineering Accuracy
- Technical specifications properly shown
- System interfaces correctly depicted
- Data flows accurately represented
- Performance characteristics realistic

#### Domain-Specific Accuracy
- Robotics concepts correctly visualized
- AI/ML concepts appropriately represented
- Humanoid-specific elements accurate
- Safety considerations properly depicted

### Accessibility Standards

#### Visual Accessibility
- Sufficient color contrast ratios
- Alternative text for complex diagrams
- Patterns as well as colors when needed
- Scalable to different sizes

#### Cognitive Accessibility
- Clear, unambiguous representations
- Logical visual hierarchy
- Consistent symbol usage
- Appropriate level of detail

### Quality Control Process

#### Diagram Review Checklist
- [ ] Technical accuracy verified by subject matter expert
- [ ] Visual clarity and readability confirmed
- [ ] Consistency with established style maintained
- [ ] Appropriate for target audience level
- [ ] Properly integrated with surrounding text
- [ ] Source and license information included when required

#### Validation Process
1. **Technical Review**: Subject matter expert verifies accuracy
2. **Style Review**: Ensures consistency with standards
3. **Accessibility Check**: Confirms accessibility compliance
4. **Integration Check**: Verifies proper relationship to text
